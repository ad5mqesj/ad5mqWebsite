The AD5MQlogger program, like many many others, has all the usual features. 
It runs on windows and will run under Cross Over Office or WINE on LINUX.

If there is enough interest, a LINUX specific version could easily be developed.  At present it uses a flat
Microsoft Access database (Access is NOT required to use it).  It has features useful for logging on field day, as well as

everyday logging, including the ability to track QSL card requests, and print reports.  The records can be edited from the

software or directly in access. Why another logging program you may ask.  Simple, none of the logging programs

I have seen on the internet or elsewhere provide a facility for people like myself who must routinely report all our foreign contacts, 

including those occurring via amateur radio.  This program addresses that need, it provides both a printed report,

 and a text file in the format  favored by all of the security officers I must report to.

 

Here is a screen shot of the program

The main interface

 

As you can see the large list control on the right shows all the records n the current database. By default these are sorted in time, newest last.

by clicking any of the column title bars in the control the contents can be sorted by that field.  the displayed list has been sorted by state and then country.

the sort order is not preserved, and will revert to default after adding or editing a record, or on restart.

Under the file menu, New will create a new database, and open will open an exiting DB.

The Setup menu contains items to add the users name, location and call sign (needed for report generation), and the report generation items.

The small button labeled "o" right of the date, allows editing the date/time field of a record.  By default new records are given the time of their creation.

If a call sign is typed into the call sign field, which already exists in the database, upon tabbing out of the field, 

all the other fields will be filled with the record from the last contact with that station.

This program is written entirely in C++ and makes extensive use of MFC. and here is the link to Download (7.4 MB)the program.

This is a self extracting archive which will extract (by default) in c:\temp.

After extracting it run the Setup.exe program which will create a minimal set of registry entries.

Then run the program and create an initial database. If the data base creation operation fails you may need the updated DAO components.

Run the included mdac_typ.exe application included with the logger program and try again.

This program has been tested on Windows 2000 and XP, Vista, and Windows 7. it may run on other platforms but "caveat emptor" applies.
